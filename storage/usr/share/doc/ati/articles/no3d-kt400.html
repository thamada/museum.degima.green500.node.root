3D Graphics acceleration not available on VIA Apollo KT400 systems

This information applies to the following:

  • ATI Proprietary Linux Driver
  • systems using VIA Apollo KT400 chipset

The system starts and 2D applications run fine, but 3D graphics will appear unusually slow.

Applications that absolutely require hardware acceleration may display an error message or simply crash.

Utilities such as glxinfo will report that Direct Rendering (hardware 3D acceleration) is not available and that the "Mesa GLX Indirect" (Software) renderer will be used instead.

This issue has been resolved in ATI Proprietary Linux Driver version 3.14 and later.
